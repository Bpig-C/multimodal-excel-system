#### 问题 2: 状态筛选不统一
**状态**: ✅ 已修复
**问题描述**:
- 前端筛选器只显示 4 个状态选项
- 实际系统使用了 7 个状态
- 状态名称在不同页面不一致
- 提交复核后状态变为 `under_review`，但前端显示为 `underreview`

**修复内容**:

1. **统一状态定义** - 创建了 `frontend/src/constants/taskStatus.ts`
   ```typescript
   export enum TaskStatus {
     PENDING = 'pending',           // 待处理
     PROCESSING = 'processing',     // 处理中
     COMPLETED = 'completed',       // 已完成
     UNDER_REVIEW = 'under_review', // 待复核
     APPROVED = 'approved',         // 已批准
     REJECTED = 'rejected',         // 已驳回
     FAILED = 'failed'              // 失败
   }
   ```

2. **更新状态筛选器** - 添加所有 7 个状态选项
   - 待处理 (pending)
   - 处理中 (processing)
   - 已完成 (completed)
   - 待复核 (under_review)
   - 已批准 (approved)
   - 已驳回 (rejected)
   - 失败 (failed)

3. **统一状态显示**
   - 更新 `AnnotationList.vue` 使用统一的状态常量
   - 更新 `DatasetDetail.vue` 使用统一的状态常量
   - 确保所有页面显示一致的状态文本和颜色

4. **状态颜色映射**
   - 待处理: 灰色 (info)
   - 处理中: 橙色 (warning)
   - 已完成: 绿色 (success)
   - 待复核: 默认色
   - 已批准: 蓝色 (primary)
   - 已驳回: 红色 (danger)
   - 失败: 红色 (danger)
