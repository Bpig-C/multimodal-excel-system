## 一、核心业务流程：从原始数据到标注结果

### 参与角色
- **管理员**：负责数据准备、任务分配、系统配置
- **标注员**：负责完成具体的标注工作
- **复核员**：负责审核标注质量
- **AI（自动）**：系统自动执行的步骤

---

### 完整流程

```
第一步：准备数据
────────────────
[管理员] 上传 Excel 文件
    → 系统解析文件，提取文字内容
    → 支持同时提取 Excel 内嵌图片，与文字关联  【↑快反】
    → 数据进入语料库，等待使用

    ─────────────────────────────────────────
    【↑快反】独立入口：导入 KF 快反事件数据
    [管理员] 上传 KF 专用格式 Excel
        → 系统解析并存入快反事件数据库
        → 可在数据查询页面查看，也可进入标注流程

    【↑QMS】独立入口：导入 QMS 不合格品数据
    [管理员] 上传 QMS 专用格式 Excel
        → 系统解析并存入不合格品数据库
        → 可在数据查询页面查看
    ─────────────────────────────────────────

第二步：组织数据集
────────────────
[管理员] 从语料库中选取需要标注的内容
    → 创建数据集，设置本次标注使用的标签规则
    → 将数据集分配给具体的标注员（可按范围分段分配）

第三步：完成标注
────────────────
[系统支持两种方式，可同时使用]

方式一：AI 自动标注
    [管理员/标注员] 点击"批量标注"按钮
    → AI 自动阅读每条文字，标出实体并连接关系
    → 如有图片，AI 同时在图片上画出识别框  【↑快反】
    → 进度条实时显示完成情况
    → 完成后等待人工复核

方式二：人工标注
    [标注员] 打开标注工作台
    → 用鼠标划取文字中的实体词语，选择类型（如"缺陷现象"、"原因"等）
    → 在实体之间连线，标注关系（如"导致"、"属于"等）
    → 如有图片，在图片上框出目标区域  【↑快反】
    → 保存后提交复核

第四步：审核复核
────────────────
[复核员] 查看标注结果
    → 对照标签规则，判断标注是否准确

    ▶ 审核通过 → 该条数据进入"已完成"状态

    ▶ 退回修改 → 填写修改意见 → 标注员重新修改 → 再次提交复核
                （可多轮循环，直到通过为止）

第五步：导出结果
────────────────
[管理员] 选择已审核通过的数据
    → 导出为标准数据格式（用于 AI 模型训练）
    → 也可导出为 Excel 格式供人工查阅  【重构新增】
```

## 二、支撑流程：标签体系管理

> 在标注工作开始前，需要配置好"标签规则"——告诉标注员（和 AI）应该标注哪些类型的实体和关系。

```
[管理员] 进入标签配置页面

配置实体类型（如：缺陷现象、原因、工序、产品型号……共 16 类）
配置关系类型（如：导致、属于、发生于……）

AI 辅助生成标签定义  【重构新增】
    → 管理员选择一个标签类型
    → AI 自动生成：定义说明、典型例子、容易混淆的类型辨析
    → 管理员审核确认后生效

[管理员] 保存为一个版本（可随时回溯历史版本）
→ 创建数据集时选择使用哪个版本的标签规则
```

## 三、扩展流程：数据查询与图谱

```
【↑快反】【↑QMS】数据查询
[用户] 进入数据查询页面
    → 按条件筛选（客户、产品、缺陷类型、时间范围等）
    → 查看 KF 快反事件列表 / QMS 不合格品记录列表
    → 查看统计图表

【↑快反（旧版图谱删除后重构）】图谱查询
[用户] 进入图谱查询页面
    → 以网络图的形式查看实体之间的关系
    → 可按数据集或 QMS 数据过滤
```

## 五、绘图建议

- **推荐格式**：带泳道的流程图，泳道：管理员 / 标注员 / 复核员 / AI自动
- **来源标注**：建议用不同颜色区分四类来源（见图例），同一色系保持一致
- **简化原则**：隐藏状态机细节，主路径保持清晰，退回修改作为回路箭头
- **可拆分**：核心标注流程单独一张图，数据查询/图谱作为另一张辅助图
