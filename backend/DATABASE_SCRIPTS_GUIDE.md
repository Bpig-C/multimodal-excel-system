# 数据库脚本使用指南

## 脚本说明

### 1. `init_db.py` - 数据库初始化（首次使用）

**用途**: 首次启动项目时使用，创建所有数据库表并插入初始数据

**功能**:
- 创建所有14个数据库表（users、corpus、images、datasets等）
- 插入默认的16种实体类型（简单配置）
- 插入默认的1种关系类型（关联）
- 创建管理员账户（用户名: admin, 密码: admin123）

**使用场景**:
- 第一次部署项目
- 数据库文件不存在时
- 需要完全重建数据库结构时

**使用方法**:
```bash
cd backend
python init_db.py
```

**特点**:
- 如果表已存在，不会重复创建
- 如果标签已存在，不会重复插入
- 如果管理员已存在，会更新密码

---

### 2. `restore_simple_labels.py` - 恢复简单标签配置

**用途**: 将标签配置恢复为简单的默认配置（16种实体 + 1种关系）

**功能**:
- **删除**所有现有的实体类型
- **删除**所有现有的关系类型
- 重新创建16种实体类型（14种文本 + 2种图片）
- 重新创建1种通用关系类型（关联）

**使用场景**:
- 标签配置被修改后，想恢复到最简单的默认状态
- 测试时需要重置标签配置
- 不需要复杂的关系类型，只用一个通用"关联"关系

**使用方法**:
```bash
cd backend
python restore_simple_labels.py
```

**注意**:
- ⚠️ 会删除所有现有标签配置
- ⚠️ 不会删除已有的标注数据，但如果标注使用了被删除的标签类型，可能会出现问题
- 建议在清空数据后使用，或确保没有重要标注数据

**简单配置内容**:
- 16种实体: 产品型号、缺陷类型、客户名称、日期时间、地点部门、数量、处置措施、产品阶段、设备名称、工艺参数、生产工艺、电子元件、人员角色、改进措施、缺陷图片、缺陷区域
- 1种关系: 关联 (relates_to) - 通用有向关系

---

### 3. `update_labels.py` - 更新为复杂标签配置

**用途**: 将标签配置更新为更细粒度的配置（16种实体 + 8种关系）

**功能**:
- **删除**所有现有的实体类型
- **删除**所有现有的关系类型
- 重新创建16种实体类型（使用英文名称，更规范）
- 重新创建8种关系类型（更细粒度的关系）

**使用场景**:
- 需要更细粒度的关系类型（如"有缺陷"、"由...引起"等）
- 需要更规范的英文标签名称
- 进行更复杂的关系标注

**使用方法**:
```bash
cd backend
python update_labels.py
```

**注意**:
- ⚠️ 会删除所有现有标签配置
- ⚠️ 不会删除已有的标注数据，但如果标注使用了被删除的标签类型，可能会出现问题

**复杂配置内容**:
- 16种实体: Product、IssueType、DefectPhenomenon、CauseCategory、SpecificCause、Process、Equipment、Material、Personnel、Customer、Supplier、Action、Time、Location、Quantity、Standard
- 8种关系: has_defect、caused_by、occurs_in、uses_equipment、uses_material、performed_by、supplied_by、resolved_by

## 其他数据库脚本

### `force_clean_db.py` - 强制清理业务数据

**用途**: 清理所有业务数据，保留用户和标签配置

**功能**:
- 删除所有标注任务
- 删除所有数据集
- 删除所有语料
- 删除所有图片记录
- **保留**用户账户
- **保留**标签配置

**使用场景**:
- 测试后需要清理数据
- 重新开始标注项目
- 数据库中有错误数据需要清理

**使用方法**:
```bash
cd backend
python force_clean_db.py
```

**注意**:
- ⚠️ 会删除所有标注数据，无法恢复
- 执行前会要求确认

---

### `clean_orphan_datasets.py` - 清理孤立数据集

**用途**: 清理没有关联语料的孤立数据集

**功能**:
- 查找没有关联语料的数据集
- 删除这些孤立数据集及其标注任务

**使用场景**:
- 数据不一致时
- 删除语料后留下的孤立数据集

**使用方法**:
```bash
cd backend
python clean_orphan_datasets.py
```

---

### `check_labels.py` - 检查当前标签配置

**用途**: 查看当前数据库中的标签配置

**功能**:
- 显示实体类型数量和列表
- 显示关系类型数量和列表
- 显示每个标签的激活状态和bbox支持

**使用方法**:
```bash
cd backend
python check_labels.py
```

---

### `create_default_version.py` - 创建默认版本快照

**用途**: 将当前的默认配置保存为版本快照

**功能**:
- 读取数据库中的所有实体类型和关系类型
- 创建版本快照（版本ID: DEFAULT_V1.0）
- 标记为默认版本和只读版本
- 设置为活跃版本

**使用场景**:
- 首次部署后创建默认版本
- 重新初始化默认版本

**使用方法**:
```bash
cd backend
python create_default_version.py
```

**注意**:
- 如果默认版本已存在，会跳过创建
- 默认版本在前端显示为"默认配置 v1.0"
- 用户可以随时切换回默认版本

## 推荐使用流程

### 首次部署
```bash
cd backend
python init_db.py
```

### 日常使用
- 不需要运行任何脚本
- 通过前端"标签配置"页面管理标签

### 需要重置标签配置
```bash
# 恢复简单配置（推荐）
cd backend
python restore_simple_labels.py

# 或使用复杂配置
cd backend
python update_labels.py
```

### 需要清理测试数据
```bash
cd backend
python force_clean_db.py
```

---

## 注意事项

1. **数据安全**:
   - 所有删除操作都是不可逆的
   - 生产环境使用前请备份数据库文件

2. **标签配置修改**:
   - 修改标签配置会影响已有标注
   - 建议使用前端"版本管理"功能保存配置快照

3. **推荐方式**:
   - 首次使用: `init_db.py`
   - 日常使用: 通过前端管理标签
   - 重置配置: `restore_simple_labels.py`（简单）或 `update_labels.py`（复杂）
