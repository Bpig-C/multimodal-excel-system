# 复核功能说明

## 概述

复核功能允许复核人员审核已标注的任务，并进行批准或驳回操作。复核人员可以在复核过程中修改标注。

## ⭐ 关键步骤：如何将标注结果用于复核

### 标注人员操作

1. **完成标注**
   - 在标注编辑器中完成实体和关系标注
   - 点击"保存"按钮保存标注

2. **提交复核** ⭐ **这是关键步骤！**
   - 点击右上角"提交复核"按钮
   - 确认提交
   - 任务状态变为"复核中"
   - 任务进入复核队列

3. **等待复核结果**
   - 如果被批准：任务状态变为"已通过"
   - 如果被驳回：任务状态变回"待标注"，可以重新编辑

### 复核人员操作

1. **查看复核任务**
   - 进入"复核管理"页面
   - 查看待复核任务列表

2. **审查标注**
   - 点击任务进入复核详情
   - 查看文本和图片标注
   - 检查标注质量

3. **做出决定**
   - **批准**：标注质量合格，点击"批准"
   - **驳回**：标注有问题，填写驳回原因后点击"驳回"

## 完整流程图

```
标注任务 → 保存 → 提交复核 ⭐ → 复核队列 → 审查 → 批准/驳回
                     ↑                              ↓
                     └──────────(驳回)──────────────┘
```

## 功能特性

### 1. 复核任务列表 (ReviewList.vue)

**路由**: `/review`

**功能**:
- 显示所有复核任务列表
- 支持按状态筛选（待复核、已通过、已驳回）
- 支持按复核人筛选
- 分页显示
- 点击任务进入复核详情页

**状态说明**:
- `pending`: 待复核
- `approved`: 已通过
- `rejected`: 已驳回

### 2. 复核详情页 (ReviewDetail.vue)

**路由**: `/review/:reviewId`

**功能**:
- 查看标注任务的完整内容
- 文本标注查看（实体高亮、关系箭头）
- 图片标注查看（整图标注、区域标注）
- 复核操作：
  - **批准**: 通过标注任务
  - **驳回**: 驳回标注任务（必须填写驳回原因）
- 复核意见输入
- 标注统计信息

**只读模式**:
- 已批准或已驳回的任务以只读模式显示
- 待复核的任务允许编辑标注

**编辑功能**:
- 复核人员可以在复核过程中修改标注
- 支持添加、编辑、删除实体
- 支持添加、删除关系
- 修改会实时保存到后端

## API接口

### 1. 获取复核任务列表
```
GET /api/v1/review/tasks
参数:
  - status: 状态筛选（可选）
  - reviewer_id: 复核人ID筛选（可选）
  - skip: 跳过记录数
  - limit: 返回记录数
```

### 2. 获取复核任务详情
```
GET /api/v1/review/{review_id}
```

### 3. 批准任务
```
POST /api/v1/review/{review_id}/approve
Body:
  - reviewer_id: 复核人ID
  - review_comment: 复核意见（可选）
```

### 4. 驳回任务
```
POST /api/v1/review/{review_id}/reject
Body:
  - reviewer_id: 复核人ID
  - review_comment: 驳回原因（必填）
```

## 使用流程

1. **标注人员提交复核**
   - 在标注页面点击"提交复核"按钮
   - 系统创建复核任务

2. **复核人员查看任务**
   - 进入复核管理页面
   - 查看待复核任务列表
   - 点击任务进入详情页

3. **复核审查**
   - 仔细检查文本和图片标注
   - 如需修改，可直接编辑标注
   - 填写复核意见

4. **做出决定**
   - **批准**: 标注质量合格，点击"批准"按钮
   - **驳回**: 标注存在问题，填写驳回原因后点击"驳回"按钮

5. **任务状态更新**
   - 批准后任务状态变为"已通过"
   - 驳回后任务状态变为"已驳回"，标注人员需要重新处理

## 组件依赖

- `TextAnnotationEditor`: 文本标注编辑器（支持只读模式）
- `ImageAnnotationEditor`: 图片标注编辑器（支持只读模式）
- `reviewApi`: 复核API封装
- `annotationApi`: 标注API封装（用于编辑标注）
- `useLabelStore`: 标签配置Store

## 注意事项

1. **权限控制**: 确保只有复核人员可以访问复核功能
2. **驳回原因**: 驳回任务时必须填写驳回原因
3. **只读模式**: 已完成复核的任务以只读模式显示
4. **实时保存**: 复核过程中的标注修改会实时保存
5. **用户ID**: 当前使用临时固定ID（1），后续需要从认证系统获取真实用户ID
